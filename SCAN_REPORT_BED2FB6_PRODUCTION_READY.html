🔬 Technical Notes for Claude Code

Why periodType=month Failed

Schwab API spec strictly enforces parameter combinations. Minute-level data (frequencyType=minute) only accepts periodType=day. Attempting periodType=month with minute frequency triggers 400 validation error.

How start_date_ms Unlocks Extended History

When you set startDate (epoch milliseconds) in the API request, Schwab API switches from period-based fetching to date-range fetching. This bypasses the 10-day period cap on minute data. You can now fetch 90-300 days of minute candles, which is critical for institutional analysis (large SMA periods like MA420, MA818).

History Window Tuning

Each timeframe has a different start_days_back value based on use case:

  • 1m/3m: 10d — Real-time arbitrage (Russian Pres outfit), tight windows
  • 5m-30m: 90d — Tactical (SSO MA500, YANG MA777), institutional daily positioning
  • 1h-4h: 300d — Strategic (ERX MA816 with leverage), long-term institutional trends
  • 1D/1W: 20 years — Multi-year macro view

Rate Limit Implication

Extended windows (300d of 4h candles) may require more API calls. But because we're fetching in bulk (one call per timeframe per ticker) and respecting 120 req/min limit, no issues observed. Scan completes in ~3 min, which is acceptable.

No Regressions Detected
