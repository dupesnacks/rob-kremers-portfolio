## v3 Architecture (FINAL)

### Core Innovation: SOS Exposure States
- **Old model:** "Kid tried apple" = 1 data point (binary)
- **New model:** "Kid looked (3/2), touched (3/5), tasted (3/12), ate (3/20)" = progression tracked

Each log captures: `[food_id, exposure_states: ["looked_at", "touched", "tasted", "ate"], timestamp]`

### Weighted Baseline Formula
```
weight = state_multiplier × recency_weight × scaling_cap × regression_penalty

state_multiplier: looked_at (0.3x) to ate (1.0x)
recency_weight: last 3 days (1.0) → 30+ days (0.1)
scaling_cap: super_safe doesn't anchor forever (scales down with exploration growth)
regression_penalty: if kid regresses (ate → looked), apply 0.8x penalty
```

### 4 Critical Fixes from Claude

1. **Super-safe scaling cap** — prevents nuggets from anchoring baseline forever
   - Formula: `0.60 / (1 + ln(num_explored_foods))`
   - So: 10 foods explored = super-safe weight drops from 0.60 → 0.23
   
2. **Deduplication in trend detection** — count distinct foods, not log events
   - Before: "apple" logged 5x = "5 sweet foods trend"
   - After: "apple" logged 5x, deduplicated = "1 sweet food + others"

3. **Variety pick with fallback** — nutritional gap + sensory distance
   - If no vegetable exists within 4.5 distance: skip variety, return 2 picks instead
   - Don't force recommendations beyond reasonable distance

4. **State multiplier rationale** — documented for post-beta tuning
   - looked (0.3) → touched (0.5) = 67% jump (significant behavioral step)
   - tasted (0.8) → ate (1.0) = 25% jump (confirms acceptance)
   - Curve is intentional: flatter at bottom (looking/touching closer), steeper at top (oral steps harder)

### Additional Insights

**Regression Handling:**
- Track `last_successful_state` from past 3 logs
- If drops below `highest_state_achieved`, apply 0.8x weight penalty
- Doesn't de-recommend, just reduces weight
- Critical for real-world use (kids regress under stress)

**Minimum Sample Size:**
- Don't activate trend detection until 5+ distinct foods logged
- Below that: baseline only, no trend language

**Trend-Aligned Bonus (Scaling):**
- Not binary (v2 had flat 20%)
- Scales with trend strength: `bonus = (trend_strength - 0.6) × 0.5`
- So: 5/7 foods = ~5% bonus, 7/7 foods = ~20% bonus

## Data Model (Final)

```
FoodLog {
  food_id: string
  kid_id: string
  timestamp: datetime
  exposure_states: ["looked_at" | "touched" | "smelled" | "tasted" | "ate"]
  parent_notes: string (optional)
}

FoodProfile (aggregated from logs) {
  food_id: string
  highest_state: string
  highest_state_multiplier: float (0.3-1.0)
  most_recent_date: datetime
  last_successful_state: string (for regression detection)
  all_logs: [FoodLog]
}

Recommendation {
  rank: "Safe Pick" | "Stretch Pick" | "Variety Pick"
  food_id: string
  food_name: string
  distance: float
  explanation: string (natural language for parents)
}
```

## CRITICAL FIX (23:26 PDT - Onboarding Case)

**Claude Code found:** Super-safe foods with 20+ onboarding entries were being crushed.

**Root cause:** `num_explored = len(food_profiles)` counted safe foods in the exploration count.

**Fix implemented:** Count ONLY truly explored foods:
```python
num_truly_explored = sum(
    1 for fid, fp in food_profiles.items()
    if fid not in super_safe_foods
    and fid not in regular_safe_foods
    and fp.highest_state in ("touched", "smelled", "tasted", "ate")
)
```

**Impact:**
- Onboarding: 20 safe foods + 0 explored = `num_truly_explored = 0`
- Safe weight: `0.60 / (1 + ln(1)) = 0.60` ✅ Full strength (was being crushed to 0.15)
- After exploring 3 foods: `0.60 / (1 + ln(3)) = 0.27` ✅ Scales down naturally

**All 5 tests still passing** after fix. Committed: `5347a7b`.
