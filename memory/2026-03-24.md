## Architectural Decisions (FINALIZED — CODE REVIEW)

### 1. **Window Parameter Pattern** ✅
- **Decision:** Conditional window parameter appending (not query string override)
- **Implementation:** `?window=institutional` when institutional, no param for default (full day)
- **Rationale:** Cleaner URL parsing, avoids overriding with `?window=full`
- **Impact:** All API endpoints (`/api/outfits`, `/api/tickers`, `/api/bias`) respect window param
- **Code location:** `backend/main.py` lines 235–244

### 2. **Snapshot-Based Momentum Calculation** ⭐ NEW
- **Pattern:** Store L/S ratio snapshots at each time window (open, midday, close, full)
- **Calculation:** `momentum_delta = close_ls - midday_ls` (requires BOTH snapshots to exist)
- **Granularity:** Per outfit (ticker=NULL → '__ALL__' sentinel)
- **Timing:** Momentum populates ONLY after close window snapshot captured
- **Code location:** `database.py` lines 335–372, `backend/main.py` line 225–230
- **Why:** Allows comparing institutional positioning drift between midday and close

### 3. **Time Window Auto-Detection & Auto-Scan** ⭐ NEW
- **Pattern:** Scheduled loop runs every 5 minutes (line 34–72 in `main.py`)
- **Logic:** Detects current time window (open/midday/close/off-market)
- **Action:** Triggers light scan automatically if window not yet scanned today
- **Deduplication:** Daily state dict prevents same-window re-scans
- **Trigger points:** 9:30 AM (open), 11:30 AM (midday), 3:00 PM ET (close/institutional)
- **Code location:** `backend/main.py` lines 34–72, lifespan context manager

### 4. **NULL to Sentinel Pattern in SQLite** ⭐ NEW (CRITICAL BUG FIX)
- **Problem:** SQLite UNIQUE constraints fail on NULL because NULL ≠ NULL
- **Solution:** Use '__ALL__' string sentinel instead of NULL for outfit-level (no ticker) rows
- **Migration:** `_migrate_scan_snapshots()` converts existing NULLs and deduplicates (line 82–105)
- **Semantics:** `ticker = '__ALL__'` means "aggregated across all tickers" for this outfit
- **Why it matters:** Makes snapshot deduplication and uniqueness work correctly
- **Code location:** `database.py` lines 82–105, 316–327

### 5. **Collapsible Sections (Expandable UI Pattern)** ⭐ NEW
- **Components:** `<ActiveProgramsSection>` and `<SnapshotsViewer>`
- **Pattern:** Each section has `isOpen` state controlled by onClick header
- **Design:** Collapsed view shows summary + "click to expand" hint
- **Data loading:** Lazy-loaded on first open (snapshots dates fetched via API call)
- **Code location:** Frontend `App.jsx` lines 190–321 (ActiveProgramsSection), 323–520 (SnapshotsViewer)

### 6. **Frontend Refresh Interval** ✅
- **Standard:** 15 seconds auto-refresh when enabled
- **Toggle:** User can switch on/off (no forced polling)
- **UX:** Clear button states (● AUTO 15s vs ○ MANUAL)

## Database Schema (Phase 2 Complete)

### scan_snapshots table (NEW — CRITICAL)
```sql
CREATE TABLE scan_snapshots (
  session_date TEXT,
  time_window TEXT,        -- 'open', 'midday', 'close', 'full'
  outfit_key TEXT,
  ticker TEXT DEFAULT '__ALL__',  -- '__ALL__' = outfit-level aggregate
  long_count INTEGER,
  short_count INTEGER,
  ls_ratio REAL,           -- long/short ratio
  total_weight REAL,       -- sum of program weights
  scanned_at TEXT,
  UNIQUE(session_date, time_window, outfit_key, ticker)
);
```
