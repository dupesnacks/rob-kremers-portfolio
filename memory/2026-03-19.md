### Architectural Decisions

1. **Two-Service Deployment Pattern**
   - Frontend: Vercel (React static)
   - Backend: Railway.app / Heroku (Python FastAPI with continuous operation)
   - Reason: Python needs persistent background tasks; can't run on serverless

2. **Data Source Strategy**
   - MVP: yfinance (free, immediate, 2–3 day intraday)
   - Production: Polygon.io (Raul-level data, real-time intraday, 5+ years history)
   - Rationale: Fast iteration with free data, upgrade path for precision

3. **Scan Design**
   - Light mode: TICKERS_LIGHT (14 tickers) for quick testing
   - Full mode: TICKERS (50+ tickers) for comprehensive analysis
   - Background task with progress tracking (done/total/ticker/timeframe)

4. **Configuration Management**
   - All outfits, tickers, affinity weights in config.py
   - Hit threshold (0.0002) centralized and configurable
   - Institutional hours (9:30–16:00 EST) hardcoded per Raul spec
   - No crossover logic in hit-counting (only for system bias banner)

5. **Database Design**
   - SQLite for simplicity (no external dependency)
   - Tables: scans, rankings, signals, tickers, outfits
   - Persistence for playback + historical analysis

### Key Insights

1. **Hit Threshold (0.02%) Works** — Penny-precision detection producing sensible signals without noise
2. **Position Weights Create Clustering** — 6-SMA exponential weighting (1→8x) naturally concentrates signals
3. **Affinity Weighting is Crucial** — 2x multiplier creates realistic clustering (NVDA+Base2, TSLA+420 dominate)
4. **System Crossovers ≠ Hit-Counting** — Raul's explicit spec: System crossovers ONLY for institutional bias banner
5. **Institutional Hours Multiplier Works** — 1.5x during market hours removes pre/after-market noise
6. **Mock Data Fallback Critical** — React can work offline; real API fills in when available

### Environment Lessons

- Python 3.14 on macOS requires venv (PEP 668 prevents system-wide pip)
- Uvicorn auto-reload works well for development (`--reload` flag)
- yfinance rate limits: ~2–3 day intraday history, manageable for 14 light-mode tickers
- Local network access: Can test from PC using Mac's local IP (192.168.1.x) before deploying to production
