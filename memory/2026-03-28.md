# Session Notes — March 28, 2026 (8:37 AM PDT)

## Goal
Train ML model on historical Raul trades X/Twitter threads. 70+ categorized case studies available in GitHub SMA-outfits repo.

## Attempt: Automated Twitter Scraping

**Options Tested:**
1. Direct X.com fetch → Blocked (bot detection)
2. Nitter mirror (nitter.net) → 200 OK but empty content extraction
3. Proposal: Copy-paste workflow (most reliable)

**Technical Findings:**
- X.com actively blocks automated requests (returns "privacy extensions" error)
- Nitter serves requests but doesn't return extractable text
- Other Twitter mirrors have same constraints
- Social media scraping at scale fundamentally blocked by design

## Architectural Decision: Accept Copy-Paste Pattern

**Why:**
- Automated social media scraping hits hard limits on all platforms
- Nitter/mirrors are fallback options that don't reliably work at scale
- Copy-paste is actually the fastest path for accurate content

**Recommended Workflow:**
1. User pastes 5-10 tweets at a time in Discord/text
2. I parse, organize by category (Precision Buy, Hard Stop, etc.)
3. I structure for ML training (Q&A pairs, JSON, markdown)
4. I attribute source links for verification
5. Repeat for all 70 threads

**Why this is better:**
- Zero blocking/detection issues
- Content accuracy guaranteed
- User review loop built-in
- Still relatively fast (70 tweets in ~14 batches)

## Key Insight
The constraint isn't technical capability — it's platform-level design. Accepting this and designing around it is faster than fighting it.
