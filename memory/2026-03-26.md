## Session Summary
Spent 2+ hours debugging CORS errors preventing Vercel frontend from calling ngrok-tunneled backend API.

## Key Insights

### CORS Middleware Behavior
- **FastAPI CORS config only loads at app startup** — code changes don't apply until uvicorn restarts
- Killed and restarted uvicorn 5+ times; each time it picks up new config
- Browser cache also masks CORS fixes — hard refresh (Cmd+Shift+R) required after each backend change

### CORS Strategy Decision
**Development/Personal Tools:** Use `allow_origins=["*"]` instead of explicit whitelist
- Simpler to maintain
- No need to update CORS list every time Vercel URL or ngrok URL changes
- Acceptable for non-public APIs

**Production:** Use explicit allowlist (what Claude recommended)
- More secure but requires updating when URLs change
- Better for public APIs

### Debugging Pattern
1. Test API directly with curl: `curl -i -H "Origin: https://..." http://localhost:8001/...`
2. Verify CORS headers present in response: `access-control-allow-origin`, `access-control-allow-credentials`
3. Test from browser console with fetch() to catch CORS preflight issues
4. If fix doesn't work after code change, **always restart uvicorn** (not just reload)
5. Browser hard refresh (Cmd+Shift+R) to clear cached responses
