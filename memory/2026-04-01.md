### Dashboard Deployment & Architecture Pattern

**Original Approach (Failed):**
- Built live auto-refreshing dashboard that calls `localhost:8001/api/tickers`
- Deployed to Vercel (robkremers.com/unfairmarket-live)
- Result: 404 errors — browsers can't access localhost from remote HTTPS sites (CORS/security)

**New Architecture (Success):**
- **Static HTML dashboard with hardcoded data tables**
- **Manual update workflow:**
  1. User requests "update dashboard with latest data"
  2. I fetch live data from localhost:8001 (no changes to trading engine)
  3. I update HTML tables with new values
  4. I redeploy to Vercel
- **Benefits:**
  - Zero changes to production code (trading engine stays untouched)
  - No risk of breaking the tool
  - Simple, transparent, auditable updates
  - Safe data isolation (tool stays completely local)
- **Trade-off:** Real-time updates → manual on-request updates

**Pattern Name:** "Offline Dashboard Pattern"
- Use when: backend can't be exposed, zero code changes allowed
- How: fetch → update file → deploy (repeat on request)
- Cost: manual refreshes instead of real-time
- Benefit: production safety
