## Architecture Decisions

### Active Programs System (NEW)
**Purpose:** Convert SMA hits into trading programs with entry/exit lifecycle

**Components:**
1. **Hit History Table** - Records all unique EOT (Equity+Outfit+Timeframe+SMA Period) hits per session
2. **Consecutive Session Detection** - Counts how many sessions same EOT appears (magnetized = 2+)
3. **Protocol Assignment** - Based on position index in outfit's SMA sequence:
   - Positions 0-2 (short SMAs) → Penny Breach (-$0.01 stop)
   - Positions 3-4 (mid SMAs) → Point Break (-$1.00 stop)
   - Position 5 (long SMA) → Candle Close (full SMA stop)
4. **Magnetized Override** - When magnetized detected, always upgrade to Candle Close protocol
5. **Lifecycle States** - active, terminated (with auto-termination for stale combos)

**Database Tables:**
- `active_programs` (ticker, outfit, timeframe, sma_period, entry_price, sma_value, stop_level, protocol, is_magnetized, consecutive_sessions, status, activated_at, terminated_at)
- `hit_history` (session_date, ticker, outfit, timeframe, sma_period, price, sma_value, side, weight)

**New Endpoints:**
- `GET /api/programs?status=active|terminated` - Returns active/terminated programs
- `GET /api/vix` - Returns VIX level + regime classification (Low/Normal/Elevated/Extreme)

### Frontend Port Configuration
**Decision:** Use `VITE_API_BASE` environment variable instead of hardcoded localhost
- **Reason:** Vite convention, allows easy deployment to different servers
- **.env file:** `VITE_API_BASE=http://localhost:8001`
- **Fallback:** Defaults to `http://localhost:8000` if not set

**Pattern:** All API calls go through `${API_BASE}${path}` in `api()` helper function
