## CRITICAL LESSON: Vercel Deployment Workflow

**Issue:** Pages kept returning 404 after I created them locally and deployed
**Root Cause:** I was committing locally but NOT pushing to GitHub before deploying to Vercel
**Why:** Vercel reads from GitHub, not your local machine. If GitHub doesn't have the file, Vercel can't deploy it.

### The Pattern That Was Happening (WRONG)
```
1. Create file locally ✓
2. git commit ✓
3. vercel deploy --prod ✗ (skipped git push!)
4. File doesn't exist on production (GitHub still has old version)
5. 404 error
6. Confusion
```

### The Correct Pattern (ALWAYS USE THIS)
```
1. Edit file locally
2. git add <file>
3. git commit -m "message"
4. git push origin main       ← THIS IS THE KEY STEP
5. vercel deploy --prod       ← Only after push completes
```

### How to Check for Unpushed Commits
```bash
git status
```

If you see:
```
Your branch is ahead of 'origin/main' by X commits
```

You have commits NOT on GitHub. Run `git push origin main` before deploying.

## Key Takeaway
**Always: commit → PUSH → deploy**

If you skip the push step, Vercel deploys what's on GitHub (the old version), not what's on your machine. This is why pages seemed to "disappear" or never appear.
