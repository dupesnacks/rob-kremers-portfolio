## 🔧 Critical Architectural Patterns Implemented

### 1. Three-Layer Deduplication System
**Problem:** Same market event counted multiple times across timeframes and outfits
- OHLC components (open/high/low/close) all matching SMA on same candle = 4 hits
- Same price event on 5m, 15m, 30m, 1h charts = 4 hits
- MA54 defined by multiple outfits hitting same price on same date = N hits

**Solution:** Tiered deduplication with weight-based selection
1. **Engine Level (OHLC):** First OHLC match wins (open > high > low > close)
2. **Cross-Timeframe (engine.py):** (ticker, outfit, sma, date) → keep highest weight
3. **Cross-Outfit (ranking.py):** (ticker, sma_period, date) → keep highest weight across outfits

**Result:** Hit count reduction 70% (6,499 → 1,964 for REPEATER_9)
- Russia Pres: 3,601 → 1,268 (65%)
- Base 2/NVDA: 3,513 → 1,254 (64%)
- Preserves signal strength via weight selection (not averaging)

**Key Code:**
```python
# Engine: OHLC dedup
for comp_name, comp_val in ohlc_vals:
    if comp_val == sma_val:
        matched_comp = comp_name
        matched_val = comp_val
        break  # first match only

# Engine: Cross-timeframe dedup (date-based hash map)
def _dedup_hits_by_date(hits: list[Hit]) -> list[Hit]:
    best: dict[tuple, Hit] = {}
    for hit in hits:
        date_str = hit.timestamp[:10]
        key = (hit.ticker, hit.outfit, hit.sma_period, date_str)
        if key not in best or hit.weight > best[key].weight:
            best[key] = hit
    return list(best.values())

# Ranking: Cross-outfit dedup (per ticker)
for hit in hits:
    date_str = hit.timestamp[:10]
    key = (hit.sma_period, date_str)  # cross-outfit key
    if key not in best or hit.weight > best[key].weight:
        best[key] = hit
```

### 2. Institutional Bias & Slope Derivation Pattern

**Fixed Issues:**
- `structural_bias` was always empty string
- Slopes were hardcoded or missing
- Price position states were static

**Pattern Established:**
```python
# Bias from system crossover analysis
struct_bias = institutional_bias.get("structural_bias", "")

# Slope derived from L/S ratio
ls_ratio = long_count / short_count
if ls_ratio > 1.1:
    slope = "Rising"
elif ls_ratio < 0.9:
    slope = "Falling"
else:
    slope = "Flat"

# Price position: closes support detection
if 0.9 <= ls_ratio <= 1.1:
    price_position = "Crossed"  # consolidation zone
elif ls_ratio > 1.0:
    price_position = "Above"
else:
    price_position = "Below"
```

**Why This Works:**
- Derivation is deterministic (no separate computation needed)
- Price position "Crossed" zone (±10% of 1:1) shows consolidation
- Structural bias ties system-level health to outfit signals

## 🎯 Key Decisions

1. **Weight-Based Selection Over Averaging**
   - Deduplicated hits keep highest weight (most confident signal)
   - Alternative: averaging would dilute institutional hours multiplier
   - Chosen approach: preserve market structure signals

2. **Cross-Outfit Dedup at Ranking Level (Not Engine)**
   - Engine only knows one (outfit, sma)
   - Ranking aggregates across outfits, detects overlaps
   - Clean separation of concerns

3. **Date-Based Hash Map Pattern**
   - Simple and performant: O(n) scan, dict keyed by (ticker, outfit, sma, date)
   - Handles timezone issues (all timestamps normalized to UTC)
   - Extensible: can add intraday windows later if needed

## ⚠️ Gotchas Learned

1. **Hit Deduplication Isn't Obvious:** Must eliminate redundancy without losing signal quality. Weight-based selection beats averaging.
2. **Timeframe Layering:** Same event appears on multiple timeframes. Date-based dedup catches this.
3. **Outfit Overlap:** Multiple outfits can define same SMA period. Cross-outfit dedup essential.
4. **Port Configuration:** Vite uses 5173/5174. Hardcode both in CORS, don't guess.
5. **YFinance Anomalies:** VIX can be negative (data bug). Always validate.
