### Architectural Pattern (Validated)

**Problem:** Browser cannot access localhost:8001 API from remote Vercel domain (CORS/security block)

**Solution:** Static HTML + embedded JSON pattern
```
User request in chat → curl localhost:8001/api/tickers → JSON data
    ↓
Edit HTML file: replace window.TICKERS_DATA array + BTC metrics + timestamp
    ↓
git commit + git push origin main
    ↓
vercel deploy --prod --token [TOKEN]
    ↓
User refreshes page → sees new data
```

**Why This Works:**
- No runtime API calls from browser to localhost (impossible)
- Data embedded at build time when I fetch it
- Each refresh is atomic (single HTML update)
- Text-match edits work reliably when exact whitespace is matched
- Deployment is instant (static site)

**Key Technical Findings:**
- HTML `<script>window.TICKERS_DATA = [...];</script>` approach is clean for embedding arrays
- Timestamp updates need exact text match: `April 1, 2026 @ 3:00 PM PDT` → `April 2, 2026 @ 7:15 AM PDT`
- BTC analysis paragraph also needs exact match for atomic updates
- Single TICKERS_DATA array (compact JSON) easier to maintain than table HTML

## Critical Design Pattern Discovered (April 2, 1:22 PM)

**Why weights don't change intraday (even with fresh scans):**

The algorithm uses a 999-candle accumulation window (~8 trading days on 5m timeframe). Between 7:15 AM and 1:00 PM, only ~70 new candles were added, but 929 of the 999 are identical data. New "hits" only count when a fresh candle's OHLC exactly matches an SMA value to the penny — this is a rare event.

**This is correct behavior:**
- Prevents noisy intraday weight shifts
- Gives stable, reliable conviction scores based on accumulated institutional footprint
- Weights change only when new exact penny matches occur OR old matches fall off the 999-candle window

**Why BTC changes more:** Uses 0.5% proximity threshold instead of exact penny match, so it recalculates and shifts more frequently.

**When to expect weight changes:**
1. New exact penny match on a fresh candle
2. Candle rolls off the back of the 999-candle window
3. Full scan vs light scan (different timeframes)
4. Next trading day (overnight/premarket shifts SMA values)

**Lesson:** Identical weights across multiple scans ≠ system broken. It means institutional positioning is stable and signals are reliable, not noisy.
