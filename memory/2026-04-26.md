### Architectural Pattern: Client-Side Metrics Export

**Problem:** Pure JS dashboard has no backend API. Automated monitoring requires headless browser that can't handle DOM rendering delays.

**Solution:** Add `window.getMetrics()` function that:
- Calculates all metrics on demand
- Returns JSON immediately (no DOM parsing needed)
- Headless browser calls it, gets data in <1s
- No additional server infrastructure required

**Code:**
```javascript
window.getAstroMetrics = function() {
  const now = new Date();
  const metrics = calcPhysics(now, state.priceHistory);
  const { signal, confidence, strength } = generateSignal(metrics);
  return {
    timestamp: now.toISOString(),
    wave: metrics.wave,
    cea: metrics.cea,
    lpi: metrics.lpi,
    // ... more fields
    status: 'live'
  };
};
```

**Applicable to:** Any client-only web app that needs external monitoring (dashboards, analytics, trading systems)
