### 3. **Timestamp Display Bug Fix**

**Problem:** BYND dashboard showed "4/17" from old HTML cache, even though data.json was current (Apr 28)

**Root cause:** `shortTime()` function only displayed time, not date. User browser cached old HTML file.

**Fix applied:**
```javascript
function shortTime(iso) {
  if (!iso) return '—';
  const d = new Date(iso);
  const month = String(d.getMonth() + 1).padStart(2, '0');
  const date = String(d.getDate()).padStart(2, '0');
  const time = d.toLocaleTimeString([], {hour:'2-digit', minute:'2-digit', second:'2-digit'});
  return `${month}/${date} ${time}`;
}
```

**Result:** Now displays `04/28 14:57:00` (full date + time)

**Files changed:**
- `/Users/rk/clawd/rob-kremers-portfolio/bynd-monitor.html`
- Committed: Apr 28, 09:15 PDT

**Lesson:** When displaying timestamps, always show date + time. Client-side caching can hide data freshness if only time is visible.
