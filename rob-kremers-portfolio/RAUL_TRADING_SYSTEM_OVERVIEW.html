🎯 What Is This Tool?

SMA Outfits Institutional Momentum Engine

A real-time scanner that detects institutional positioning across stocks, ETFs, and crypto by analyzing how price moves across SMA (Simple Moving Average) "outfits" — specific sequences of moving average periods that institutional traders use as entry/exit triggers.

Core Mission: Identify which "outfit" (institutional playbook) is currently gaming a ticker, and score the strength of that positioning by counting weighted hits across timeframes.

Real-World Example:

TSLA on TSLA (420) outfit:

  • Outfit definition: [27, 53, 105, 210, 420] SMAs (geometric sequence: a(n)=420×2^n)
  • If TSLA price is currently above MA420 and approaching MA210 from below = institutional accumulation signal
  • The scanner counts this as a "hit" for TSLA (420) outfit on that timeframe
  • Across all timeframes (1m, 3m, 5m, 10m, etc.), total hits = conviction strength

What You Get: Live feed of institutional trades before retail sees them, ranked by confidence and timeframe.

2. Hits & Weights

A "hit" occurs when price crosses an SMA for that outfit. Example:

  • TSLA price: $250
  • TSLA (420) MA210: $249
  • → Price above MA210 = HIT (accumulation signal)
  • Weight = score based on outfit strength + affinity boost (2× if historical match)

Total Weight = Sum of all hits for a ticker across all outfits/timeframes. Higher total = stronger institutional positioning.

3. Timeframes & Data

Scanner checks multiple timeframes per ticker:

  • 1m, 3m, 5m, 10m, 15m, 20m, 30m: Intraday (institutional hour)
  • 1h, 4h: Swing trade
  • Daily, Weekly: Trend confirmation

Data Source: Schwab API (real-time, <5s latency) + Coinbase (BTC public API)

4. Affinity & Historical Emphasis

If a ticker has a known affinity for an outfit (e.g., TSLA→TSLA/420), hits on that outfit get a 2× score boost.

Why? Historical observation shows TSLA often operates on 420 outfit, but it's not restricted to it anymore (see "Recent Changes" below).

⚙️ How It Works (Full Data Pipeline)

Step 1: Fetch OHLC Data

schwab_fetcher.py uses Schwab OAuth2 to fetch price history. Respects 7-day token expiry (requires manual re-auth every 7 days).

Rate Limit: 120 req/min. Scan adapts to throttling.

Step 2: Calculate SMAs

For each outfit + timeframe, compute moving averages for all periods in the outfit definition.

Example: TSLA (420) on 5m timeframe → calculate MA27, MA53, MA105, MA210, MA420 from 5m OHLC candles

Step 3: Detect Hits

Compare latest close price to each SMA. If price crosses (breaks above or below), count as HIT.

Hit Logic:

  • Price > SMA = LONG hit (buy signal)
  • Price < SMA = SHORT hit (sell signal)
  • Weight = 4.0 (critical), 2.0 (support), 1.0-1.5 (minor) depending on SMA tier

Step 4: Affinity Boost

is_affinity(outfit, ticker) checks if ticker has historical match. If yes, multiply weight × 2.

Example: TSLA hit on TSLA (420) = 4.0 weight × 2 = 8.0 effective weight

Step 5: Rank by Total Weight

Sum all hits across outfits + timeframes for each ticker. Sort by total weight. Return top 31 (light mode) or all (full mode).

Output: `/api/tickers` endpoint with ranked list

💡 Key Patterns Discovered

1. Extended History Windows Reveal True Conviction

Pattern: 90-300 day history shows sustained institutional positioning; 10-day windows show hedging noise.

  • TSLA 10-day: 1.90x L/S ratio
  • TSLA 90-300d: 2.45x L/S ratio (true conviction)

Rule: Always interpret short-term shifts through long-term lens.

2. BTC Risk Regime Controls TSLA Rebalancing

Pattern: BTC support/resistance breaks trigger institutional rebalancing in TSLA instruments.

  • BTC breaks $72k resistance → TSLA spot flips bullish
  • BTC tests $70.4k support → TSLQ bearish strengthens

Lead Time: BTC moves first by 4-24 hours; TSLA instruments follow.

3. Spot vs Leverage Entry Ladder

Pattern: Institutions enter via leverage FIRST (conviction building), confirm via spot accumulation SECOND (macro trigger).

  • Week 1-2: Accumulate in TSLL/TSLR/TSLT while maintaining TSLQ hedge
  • Week 3+: Wait for macro confirmation (BTC risk-on breakout)
  • Confirmation: Spot TSLA accumulation activates, hedge ratio optimized

4. Hedge Ratio Optimization ≠ Exit

Pattern: Institutions reduce hedge allocation on macro confirmation, not to signal conviction loss.

  • Reduction on breakout = bullish (capital reallocation)
  • Increase on weakness = bearish (risk-off)
  • Static hedge = neutral (position holding)

Example: TSLQ reduced 1,849w → 1,736w (-112.5w) while net TSLA long increased from +2,415w to +2,509w.
