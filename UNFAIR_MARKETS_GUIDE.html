Running Scans

Light Scan (23 Tickers — Default)

curl -s -X POST 'http://localhost:8001/api/scan?mode=light'
# Response: {"status": "started", "mode": "light"}
# Wait 2-3 minutes for completion

Full Scan (79 Tickers)

curl -s -X POST 'http://localhost:8001/api/scan?mode=full'
# Wait 5-9 minutes for completion (much longer)

Check Scan Status

curl -s 'http://localhost:8001/api/scan/status'
# Shows: in_progress: true/false, progress with ticker/timeframe

Fetch Results (After Scan Completes)

curl -s 'http://localhost:8001/api/tickers'
# Returns: JSON array of all scanned tickers with weights, L/S ratios, key SMAs

curl -s 'http://localhost:8001/api/btc'
# Returns: BTC specific endpoint with price, verdict, long/short weights

Extract Specific Tickers

curl -s 'http://localhost:8001/api/tickers' | jq '[.[] | select(.ticker | test("^TSLA|^TSLL|^TSLQ"))]'
# Returns: TSLA family data only

Updating the Dashboard

Step 1: Get Fresh Data

curl -s 'http://localhost:8001/api/tickers' > /tmp/tickers.json
curl -s 'http://localhost:8001/api/btc' > /tmp/btc.json

Step 2: Extract Key Metrics

# Top 23 tickers (already formatted in JSON)
jq '.' /tmp/tickers.json

# BTC summary
jq '.price, .verdict, .long_weight, .short_weight' /tmp/btc.json

Step 3: Edit HTML File

Open /Users/rk/clawd/rob-kremers-portfolio/unfairmarket-live/index.html

  5. Analysis Box: Update "Overall Institutional Positioning" with fresh interpretation

Interpretation Rules (Quick Reference)

Metric What It Means Action
Weight >500w Extreme conviction Track closely, likely to move fast
L/S > 3.0x Bullish aggression Buy dips, resistance levels key
L/S < 0.5x Bearish aggression Watch for liquidation cascade
L/S = 1.0x Balanced/hedged Profit-taking or uncertainty
Rank #1-5 Top institutional bets Core exposure, likely to hold
Dominant Outfit Which SMA cluster Key support/resistance at those periods
Slope: Rising Momentum up Institutions accumulating
Slope: Falling Momentum down Institutions distributing

Common Workflows

"Update with fresh scan results"

  1. curl -X POST 'http://localhost:8001/api/scan?mode=light' (start scan)
  2. Wait 2-3 minutes
  3. curl 'http://localhost:8001/api/tickers' > /tmp/tickers.json (fetch)
  4. Extract key data, update HTML file (see Step 3 above)
  5. git commit + vercel deploy (deploy)

"Check current BTC status"

  1. curl 'http://localhost:8001/api/btc' (no scan needed, uses last scan results)
  2. Extract price, verdict, weights
  3. Share raw JSON if detailed analysis needed

"Track specific ticker (e.g., TSLA family)"

  1. curl 'http://localhost:8001/api/tickers' | jq '[.[] | select(.ticker | test("^TSLA"))]'
  2. Shows all TSLA tickers (light scan has ~3, full scan has 5)

Critical Notes

❌ Do NOT:

  • Edit dashboard directly via Vercel UI (changes revert on next deploy)
  • Forget to git commit before deploying (old code redeploys)
  • Run full scan unless needed (takes 5-9 minutes vs light 2-3)
  • Deploy without fresh git add/commit (GitHub version is old)

✅ Always:

  • Check curl 'http://localhost:8001/api/scan/status' before fetching (ensure scan finished)
  • Keep /tmp/tickers.json and /tmp/btc.json as backups
  • Update timestamp to exact time you ran scan (for clarity)
  • Maintain consistent narrative style in analysis paragraphs (factual, pattern-focused, conviction-based)
  • Verify Vercel deploy with "Aliased: https://www.robkremers.com" message
